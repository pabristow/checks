Preface

Checks are required on alphanumeric identifiers in numerous domains such as the distribution chain (product bar codes), banking (bank account, credit cards, fidelity cards, ...) and many others. These codes and numbers are often typed, copied or scanned by humans or machines; and both make errors. We need a way to detect errors and this is why check digits have been designed.

A check digit aims to control the validity of a alphanumeric string and catch the as many changes as possible. (types of error discuss possible alterations and the efficiency of their detection).

This library provides a collection of functions for validating and creating check digits.

Scott McMurray has identifed four fairly distinct types of check:

  1. ISBN/ISSN/UPC/EAN/VISA/etc, for catching human-entry errors.
  2. hash functions as in hash tables, which only care about distribution.
  3. checksums like CRC32, for catching data transmission errors.
  4. and cryptographic hash functions, the only ones useful against malicious adversaries.

The functions in this Boost.Checks library are primarily for the first category : catching human-entry errors (though it obviously also provides against a mis-scan or mis-transmit by a device like a bar code or card reader.)

This library supports four families of check : Modulus 10, Modulus 11, Modulus 97-10 and Verhoeff. A lot of other check systems are inherited from these families.
